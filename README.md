### 工作任务

#### 自习室小程序（2人）

- 实现一个自习室小程序，供**学生**使用
- 显示百度摄像头实时传输的自习室人数（百度方面会提供相应接口，具体联系周文杰老师）
- 截止时间：==**2月3日23:59**==，之后加入地下室预约小程序组，后续具体安排视预约组进展而定。

#### 地下室预约小程序（4人）

- 实现一个小程序，供**学生**和**地下室管理员**使用
- 预约程序由腾讯实现，我们不用实现预约功能，我们的小程序只负责房间使用结束后的确认（即使用房间需要用到<u>腾讯提供的</u>和<u>我们开发的</u>两个小程序，一个预约，一个）
- 我们可以访问腾讯预约小程序的数据库（大概），可以访问预约的时间信息；同时在腾讯微校上，我们可以限制腾讯小程序的访问权限，只对部分学生开放
- 功能一：**【确认程序】**（2人）
  - <u>学生使用完房间</u>后（这里需要访问腾讯预约小程序的数据库），给学生发送消息提醒其【确认】，<u>学生确认后</u>给管理员发消息提醒其【确认】
  - 管理员确认，才算完成一次房间的使用
  - 截止时间：==**2月10日23:59**==（大概，实现基本功能，具体情况视腾讯的数据库接口而定）

- 功能二：**【信用系统】**（2人）
  - 建立数据库，记录学生违约情况
  - 违约的判定：预约房间但未及时到场/未在规定时长内确认使用完毕/物业判定违规使用
  - 一学期内违约累计三次者暂停其预约服务7天，7天后清空之前的违约记录
  - 暂停服务的方式：<u>我们的小程序</u>在腾讯微校将<u>腾讯预约小程序</u>用户权限设置为该学生不可访问。
  - 截止时间：==**2月10日23:59**==（大概，实现基本功能，集体情况视腾讯的数据库接口而定）
